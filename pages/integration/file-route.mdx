# File route

This endpoint is used to upload files per brand.

## Request
- Method `POST`
- Header `Authorization`
- Route `client-files`

Body contains array of file and array of data. Files are multipart/form-data. Data is json object.
```text
Array of files
```
```json
{
    "schema": {
      "type": "object",
      "properties": {
        "files": {
          "type": "array",
          "items": {
            "type": "string",
            "format": "binary"
          }
        },
        "data": {
          "type": "array",
          "example": [
            {
              "brandId": 1,
              "tableName": "user"
            },
            {
              "brandId": 1,
              "tableName": "user"
            }
          ]
        }
      }
    }
  }
```

## Response
- 200 OK - Success
```json
{
    "statusCode": 200,
    "message": "Files Uploaded Successfully"
}
```

- 206 Partial success - If there were errors regarding JSON format, files with errors will be skipped and their list will be in the response message
```json
{
  "statusCode": 206,
  "message": "Files uploaded partially, skipped files"
}
```

- 400 Bad Request
```json
{
  "statusCode": 400,
  "error": "Bad Request",
  "message": "Error while removing all client tables for clientId 123: YourErrorMessage"
}
```
- 401 Unauthorized
```json
{
    "statusCode": 401,
    "message": "Please provide a token!",
    "error": "Unauthorized"
}
```
- 500 Server Error - Internal server error while syncing the dbs
```json
{
  "statusCode": 500,
  "error": "Server error",
  "message": "Server error message"
}
```
